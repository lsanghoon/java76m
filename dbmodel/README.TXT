[DB 모델링]
1. 개요
- 데이터가 중복 ==> 데이터가 일치하지 않는 문제 ==> 업무에 문제 발생
- 중복 데이터를 제거 ==> 데이터를 일치시킨다. ==> 무결성 유지
- 용어
  key : 데이터를 구분하기위한 컬럼들의 집합
  candidate key(후보키, 최소키) : 키 중에서 최소 후보자를 골라낸 것
  primary key : DBMS 관리자가 후보키 중에서 선택한 키.
  entity : 테이블
  record : 테이블에 저장된 한 데이터 = row
  column : 테이블의 한 속성 = attribute

2. 정규화
1) 제1 정규화
   - 중복 컬럼이나 데이터가 있다면 별도의 테이블로 분리하고 부모-자식 관계를 맺는다.
   
2) 제2 정규화
   - 주 키가 여러 개의 컬럼으로 구성되었을 때, 나머지 모든 일반 컬럼은 주키 컬럼 모두에 종속되어야 한다.
     만약 종속되지 않는 일반 컬럼이 있다면별도 테이블로 분리하여 부모-자식 관계를 맺는다.
     
3) 제3 정규화
   - 주 키가 단일 컬럼으로 구성되었을 때, 주 키에 종속되지 않고 다른 일반컬럼에 종속된 데이터가 
     있다면 별도 테이블로 분리하여 부모-자식 관계를 맺는다.
     
3. 논리 모델을 물리 모델로 전환
1) 다 대 다 관계를 해소한다.
  - 관계를 나타내는 테이블을 생성한 후 부모-자식 관계를 맺는다.

2) 일반 용어로 된 테이블 및 컬럼 이름을 DBMS에 맞게끔 변경한다.
  - first name ==> FRT_NM

3) 컬럼의 도메인을 정의한다.
  - 컬럼들을 유사한 타입으로 묶어 분류한다. = 일관성있게 컬럼의 타입을 관리할 수 있다.  

4. 인덱스 컬럼을 지정한다.
1) 후보키(candidate key) 컬럼을 유니크(unique) 컬럼으로 지정한다.
2) 검색할 때 조건으로 자주 사용하는 컬럼은 인덱스 컬럼으로 지정한다.

5. 포워드 엔지니어링 수행
  - 모델링 결과를 가지고 SQL문을 자동 생성한다.
  - 참고: 리버스 엔지니어링(reverse engineering)
         => DBMS 테이블 정보를 읽어 모델링을 생성
